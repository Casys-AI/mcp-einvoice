GET     /v1.beta/companies/me
        Tags: companies
        Summary: Show the current company
        Description: Show the company associated with the current access token.
        200 Returns (application/json): company

GET     /v1.beta/directory_entries
        Tags: directory_entries
        Summary: List directory entries
        Description: List the directory entries of the company.

The response contains addresses that can be used to send an invoice to the company.


        200 Returns (application/json): list_directory_entries

        Tags: directory_entries
        Summary: Create a new directory entry
        Description: Create a directory entry
        Body (application/json): create_directory_entry
        200 Returns (application/json): directory_entry

DELETE  /v1.beta/directory_entries/{id}
        Tags: directory_entries
        Summary: Delete directory entry
        Description: Delete a directory entry. "Reply-to" addresses are technical addresses and cannot be deleted
        Params: id

GET     /v1.beta/directory_entries/{id}
        Tags: directory_entries
        Summary: Get a directory entry
        Description: Get directory entry
        Params: id
        200 Returns (application/json): directory_entry

GET     /v1.beta/invoice_events
        Tags: invoice_events
        Summary: List invoices events
        Description: List invoices events.

 By default, events for all invoices are listed. Use `invoice_id` parameter to select only events related to one invoice.

Invoice events are sorted by id in ascending order.

Pagination is controlled with 2 parameters: `limit` and `starting_after_id`. These parameters allows 
        Params: invoice_id, starting_after_id, limit
        200 Returns (application/json): list_events

POST    /v1.beta/invoice_events
        Tags: invoice_events
        Summary: Create an invoice event
        Description: Create an invoice event message and place it in a queue to be sent asynchronously. If the status_code is 'fr:212' without details, the amount will be the invoice total.
        200 Returns (application/json): event

GET     /v1.beta/invoices
        Tags: invoices
        Summary: List invoices
        Description: List invoices.

By default, both incoming and outgoing invoices are listed. Use the `direction` parameter to select only incoming or outgoing invoices.

The list of invoices can be narrowed down to a period with the `date` parameter. Accepted format are `YYYY`, `YYYY-MM`, `YYYY-MM-DD`.
